# Lab 3.1: Component with State - Advanced Counter

## Описание проекта

Этот проект реализует компонент счетчика с расширенной функциональностью, демонстрирующий использование React hooks (`useState`) и работу с props и state.

## Структура компонентов

### StepCounter
Основной компонент счетчика, который:
- Принимает props: `initialValue` (начальное значение) и `step` (шаг изменения)
- Управляет тремя состояниями:
  - `count`: текущее значение счетчика
  - `history`: массив всех значений счетчика
  - `operationCount`: количество выполненных операций
- Отображает статистику и последние 5 значений из истории

### CounterApp
Родительский компонент, который:
- Рендерит два независимых экземпляра `StepCounter`
- Первый: `initialValue={0} step={1}`
- Второй: `initialValue={10} step={5}`
- Каждый счетчик работает независимо со своим состоянием

## Как запустить проект

1. Установите зависимости:
```bash
npm install
```

2. Запустите проект в режиме разработки:
```bash
npm start
```

3. Откройте [http://localhost:3000](http://localhost:3000) в браузере

## Объяснение Props vs State

**Props (Свойства):**
- Передаются от родительского компонента к дочернему
- Являются неизменяемыми (immutable) внутри компонента
- Используются для конфигурации компонента
- В данном случае: `initialValue` и `step` задают начальные параметры счетчика

**State (Состояние):**
- Управляется внутри компонента с помощью `useState`
- Может изменяться через функции обновления состояния
- Изменения состояния вызывают перерисовку компонента
- В данном случае: `count`, `history`, `operationCount` отслеживают динамические изменения

**Взаимодействие:**
Props предоставляют начальную конфигурацию (например, `initialValue` используется для инициализации `count`), а state отслеживает изменения во время жизненного цикла компонента.
